  1. Classical Solution vs Quantum Solution

Classicaly, Hidden Boolean function f(x) maps n-bit inputs to either 0 or 1.

(1)f:{0,1}n−>{0,1}

The goal of f(x) is to know if it's balanced or constant. If we get both 0 and 1, then we know the function is balanced as we have obtained the two different outputs. In the worst case, if we continue to see the same output for each input we try, we will have to check exactly half of all possible inputs plus one in order to be certain that f(x) is constant. ($2^n-1$ steps).

However, using a quantum computer, we can solve this problem with 100% confidence after only one call to the function f(x), provided we ahve the function f implemented as a quantum oracle, which maps the state $\vert x\rangle \vert y\rangle $ to $ \vert x\rangle \vert y \oplus f(x)\rangle$.

Q. How it works?

A. In the quantum algorithm, we call the oracle once on a superposition of all 2^n possible inputs. Thus, the infromation about f(x) for all inputs is effectively probed in a single oracle invocation.

  1. Oracle

Oracle is a kind of function and its name is derived from the divine message of ancient Greek myth.

Oracle can take any input such as numbers, characters but only return logical value 0 or 1. When dealing with oracle, we cannot know the exact process of the oracle. That is, the only thing we could know is just the output(0 or 1) of them. Therefore, we can call it a 'Black Box model'.

Every operation in the quantum computer is done by Unitary Matrix. Therefore, oracle is also made my unitary matrix $U_f$. Oracle is made like below.

(2)Uf(|x⟩)={−|x⟩,if f(|x⟩)=1|x⟩,if f(|x⟩)=0

Oracle flips the amplitude of the result. According to result of measurement after oracle, we can know if it's constant or balanced. Constant is when it has only 0, balanced is when it has anything else but only 0.
